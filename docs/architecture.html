architecture overview

onnx2oracle

How HuggingFace embedding models get welded into Oracle AI Database as single-graph ONNX, uploaded through DBMS_VECTOR, and queried via VECTOR_EMBEDDING.

// contributor-facing doc (not the user guide)
Python 3.10+ Typer CLI onnx onnxruntime onnxruntime-extensions transformers huggingface-hub oracledb (thin) Oracle 26ai Free

Modules

cli.py
Typer commands: load, verify, presets, docker, config. Thin orchestrator that wires the other modules together.
deps: typer, rich
presets.py
Frozen ModelSpec dataclasses for 6 curated embedding models. Knows dims, size, pooling strategy, HF repo ID, and Oracle model name per preset.
deps: stdlib only
connection.py
Resolves a DSN by checking CLI args, env vars, a toml file, the --target local shortcut, and finally an interactive prompt. Returns an OracleConnection record.
deps: oracledb, tomllib
pipeline.py
The interesting bit. Downloads an HF model, wraps its fast tokenizer as ONNX ops, merges graphs, appends pooling + L2 norm, emits a single string-to-vector graph.
deps: onnx, onnxruntime-extensions, transformers, huggingface-hub
loader.py
Uploads the augmented ONNX bytes via DBMS_VECTOR.LOAD_ONNX_MODEL over a thin oracledb session. Handles chunked transfer for models larger than 500 MB.
deps: oracledb
verify.py
Smoke test: runs VECTOR_EMBEDDING against a loaded model, checks shape + L2 norm, confirms king/queen > king/banana on cosine similarity.
deps: oracledb

How the augmented ONNX is built

Oracle's VECTOR_EMBEDDING needs a single self-contained graph that goes string in, float32 vector out. HuggingFace ships a Python tokenizer separate from the model. pipeline.py welds them together in 8 stages.

download
Pull the model repo from HuggingFace Hub (config + tokenizer + weights).
tokenizer as ONNX
Emit the fast tokenizer as ONNX ops using onnxruntime-extensions.
opset align
Normalize opset versions between tokenizer graph and transformer body so onnx.compose.merge_models accepts both.
Unsqueeze
Add a batch dimension on token ids so the transformer input shape lines up.
merge
Compose tokenizer + transformer into one graph. IO: string -> last_hidden_state.
pool
Append mean-pooling or CLS-pooling (from the ModelSpec) across the sequence axis.
L2 normalize
Bolt an L2 normalization node on top so downstream cosine similarity works correctly.
Squeeze
Drop the batch dim. Output is a 1-D float32 vector of the preset's declared size.
